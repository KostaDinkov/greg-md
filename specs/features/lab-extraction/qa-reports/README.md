# QA Reports Directory

This directory contains validation reports generated by the `qa-validator` agent.

## Purpose

Each QA report documents:

- Test execution results (backend, frontend, E2E)
- Acceptance criteria validation status
- Defects found (if any)
- Go/No-go decision
- Evidence and recommendations

## Naming Convention

Reports are named: `{YYYY-MM-DD-HHMM}-qa-report.md`

Example: `2026-05-16-1630-qa-report.md`

## Usage

Generate a QA report using:

```
/test-feature lab-extraction
```

Reports are automatically saved here by the test-feature workflow.

## Report Lifecycle

- **Initial Validation**: First QA run on completed implementation
- **Re-validation**: After fixing NO-GO defects
- **Final Validation**: Last QA before reviewer sign-off

Keep all reports for audit trail and historical reference.
